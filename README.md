# 应用 icon 管理端

一个轻量的本地管理端，用于统一维护**应用图标**与 **SVG 符号库**，支持增删改查、搜索分类筛选、上传预览、导入导出。

## 技术栈

- 后端：Node.js + Express
- 数据库：SQLite（better-sqlite3，单文件免安装）
- 文件存储：本地 `uploads/` 目录
- 前端：原生 HTML / CSS / JS（无构建步骤）

## 快速开始

```bash
# 1. 安装依赖
npm install

# 2. 启动服务
npm start

# 3. 浏览器访问
http://localhost:3000
```

默认端口 3000，可通过环境变量覆盖：`PORT=8080 npm start`。

## 功能说明

| 功能 | 说明 |
|------|------|
| 新增图标 | 填写名称、类型、分类、标签、版本、描述，上传图片或 SVG（≤5MB） |
| 编辑 / 删除 | 卡片下方操作；编辑时不选文件则保留原图，删除同时清理磁盘文件 |
| 搜索筛选 | 顶部按名称/标签/描述搜索，按类型、分类下拉筛选 |
| 预览 | 点击缩略图放大查看，可复制名称、复制链接、下载 |
| 导入 / 导出 | 导出全部数据为 JSON 备份；导入 JSON 追加合并 |

## 数据与目录

```
data/icons.db      SQLite 数据库（运行时自动生成）
uploads/           上传的图标文件（运行时自动生成）
public/            前端页面
server.js          后端服务入口
db.js              数据库初始化
```

> 数据持久化在 `data/icons.db`，文件存于 `uploads/`。备份时一并保留这两处即可。
> 导出的 JSON 仅含元数据与文件路径引用，完整备份请同时保留 `uploads/` 目录。

## 后续维护

- 新增图标：直接在界面操作即可，无需改代码。
- 修改字段或类型：调整 `db.js` 表结构与 `server.js` 对应字段。
- 端口、文件大小限制等：见 `server.js` 顶部常量。
